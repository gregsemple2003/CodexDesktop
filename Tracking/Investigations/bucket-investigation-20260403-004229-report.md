## Avoidance
- Do not launch fresh pass workers from a bloated long-lived parent with `fork_context:true`; either continue in the existing leader or spawn a fresh worker with a minimal disk-grounded prompt and `fork_context:false`.
- Do not treat “autoapprove UI work for the rest of this thread” as permission to autonomously chain multiple new pass leaders from the same swollen supervisor thread. Stop at pass boundaries unless the user explicitly asks for the next pass.
- If a supervisor must watch a child, stop the 30-90 second `wait_agent` plus child-log tail loop. Use longer waits or completion notifications, and only inspect logs when there is evidence of a stall.
