# Problem 0005 Option C Plan 0001

## Intent

Always end milestones with a next-step menu for the human to pick from.

## What Changes

- Add a short options list at each checkpoint.
- Let the human choose whether to continue, pivot, or stop.
- Use the menu to make coordination more explicit.

## Files Or Artifact Types That Move

- Milestone message templates.
- Optional handoff notes that list the next-step options.
- Shared workflow guidance for option menus.

## Rollout

1. Define a short standard menu shape.
2. Use it at checkpoints where multiple legitimate next steps exist.
3. Pilot the pattern on planning-heavy tasks.
4. Check whether clarity gains outweigh the extra human decision cost.

## Success Check

- The menu is short and actionable.
- Humans understand the current decision point faster.
- Menu use stays limited to real forks.

## Burden Reduction Under Directional Context

`Truth`: it can make branch points clearer.

`Compassion`: it may reduce some confusion, but it still pushes a choice back to the human.

`Tolerance`: it helps when the human wants explicit control over the next move.
