# Bug 0001

## Summary

Task-level regression is blocked on proving the real global hotkey interaction.

## Current Evidence

- the real desktop app launches
- the overlay renders and exports artifacts
- the ingest path works against the live `.codex` tree
- the smoke summary for `REGRESSION-RUN-0001` shows:
  - `hotkey_triggered=False`
  - `overlay_fallback=True`

## Why This Matters

The task acceptance criteria explicitly include a global hotkey that can show and hide the overlay while the app remains backgrounded.

The current automated smoke path does not prove that claim yet.

## Likely Cause

The synthetic keyboard-input path used by the smoke harness does not trigger the registered global-hotkey callback in this environment.

This is currently treated as a regression-proof gap, not yet as a product defect in the interactive hotkey path itself.

## Next Step

Use a real human keyboard interaction to verify the hotkey path once:

1. launch `python -m app.codex_dashboard`
2. press `Ctrl+Alt+Space`
3. confirm the overlay appears
4. dismiss it with the same hotkey or `Escape`

If that works, close this bug as a harness limitation and update the regression run with the real interaction result.
