# BUG-0003: Completed Tasks Display As Waiting On You

Status: closed

## Summary

The `Tasks` tab showed completed tasks under `Needs you` with visible
`Waiting on you` copy.

Example observed from the human dashboard window:

- visible selected row: `Task-0001`
- visible label: `Waiting on you`
- visible summary tile: `Needs you = 11`

Durable truth for `Task-0001`:

```json
{
  "status": "complete",
  "phase": "closure",
  "current_gate": "closure",
  "validation": {
    "unit_tests": "passing",
    "regression": "passing"
  }
}
```

## Impact

- The dashboard assigns non-existent work to the human.
- Completed work looks active and urgent.
- The task summary count becomes untrustworthy.
- A dashboard regression that only proves launch/API readiness can pass while
  the product surface is semantically false.

## Evidence

Backend readback returned:

- `state_envelope.state = blocked`
- `reason_code = task_state_unmapped`
- `attention.attention_level = needs_attention`

for `Task-0001`, even though `Tracking/Task-0001/TASK-STATE.json` records
`status = complete`.

Likely root-cause hypothesis:

- backend task-state mapping recognizes `done`, `completed`, and `closed`
  as complete statuses
- the shared task-state schema uses `complete`
- therefore `complete` falls into the backend unmapped/default branch
- frontend maps `needs_attention` to `Waiting on you`

## Required Fix

- Backend task-state reconciliation must recognize the schema value
  `status: complete` as completed.
- Completed tasks must not be dispatch-ready and must not appear in `Needs you`.
- The frontend must not translate generic `needs_attention` into `Waiting on you`
  unless the durable owner is actually human.

## Required Regression

Add or update `REG-004 Semantic Dashboard State Reconciliation` evidence with a
matrix row proving:

- durable source: `Tracking/Task-0001/TASK-STATE.json`
- durable truth: `status = complete`, `phase = closure`
- backend state: `completed`
- visible dashboard state: not `Waiting on you` and not counted in `Needs you`

Do not close this bug with API proof alone. Closure requires app-surface
semantic reconciliation.

`Waiting on you`, no longer count in `Needs you`, and no longer expose
dispatch/run-control actions on the active Tasks surface.
