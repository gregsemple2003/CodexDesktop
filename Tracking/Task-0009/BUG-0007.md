# BUG-0007: Duplicate Open Task Buttons Have No Observable Effect

Status: open

## Summary

The selected task detail pane can show three adjacent buttons with the same
visible label, `Open Task`, and the human reports that none of them do
anything.

## Visible Evidence

Human-provided dashboard screenshot on 2026-04-27:

- selected task: `Task-0011`
- visible action row: `Dispatch`, `Open Task`, `Open Task`, `Open Task`
- the three `Open Task` buttons are indistinguishable
- reported behavior: clicking them has no observable effect

## Why This Is False Or Unsupported

The action row is supposed to expose bounded, understandable task navigation.
Three identical labels do not tell the human which artifact or context each
button opens. If the click path silently fails, the dashboard is also presenting
a command surface that does not execute the advertised command.

## Likely Root Cause

The frontend appears to map multiple backend artifact or launch targets to the
same fallback label, `Open Task`, and either the launch handler is not receiving
a usable target URI or failures are not visible on the surface.

## Expected Behavior

- Each visible action has a distinct, truthful label such as `Open Task`,
  `Open Plan`, `Open Handoff`, or `Open Working Context`.
- Each enabled open action launches the intended target or shows a visible,
  actionable failure reason.
- Duplicate labels are avoided unless the duplicate actions genuinely do the
  same thing and only one button is shown.

## Required Closure Proof

- Actual dashboard-surface proof shows the selected planning task no longer has
  three indistinguishable `Open Task` buttons.
- Clicking each visible open action produces the expected editor/thread/context
  launch or a visible failure state.
- Regression evidence records the visible action labels and their target
  results for a task with `TASK.md`, `PLAN.md`, and `HANDOFF.md` artifacts.
