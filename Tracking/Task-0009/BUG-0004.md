# BUG-0004: Cancelled Tasks Display As Active Work

Status: closed

## Summary

The `Tasks` tab presents cancelled tasks as active committed work in
human-facing buckets.

In the `2026-04-27` isolated validation-lane REG-004 run:

- `Task-0004` durable state is `status = cancelled`, `phase = closure`
- `Task-0006` durable state is `status = cancelled`, `phase = closure`
- both tasks were returned through `/api/v1/tasks`
- frontend mapping placed both tasks in the active `needs_you` bucket
- `Task-0004` was also exposed as dispatch-ready

## Durable/Backend Evidence

Durable sources:

- [Task-0004 TASK-STATE.json](../Task-0004/TASK-STATE.json)
- [Task-0006 TASK-STATE.json](../Task-0006/TASK-STATE.json)

Both durable state files record terminal cancelled status:

```json
{
  "status": "cancelled",
  "phase": "closure"
}
```

Backend evidence from `/api/v1/tasks`:

- `Task-0004` returned `state_envelope.state = blocked`,
  `reason_code = task_state_unmapped`, `next_owner = backend`, and
  `dispatch_readiness.ready = true`.
- `Task-0006` returned `state_envelope.state = waiting_for_human`,
  `next_owner = human`, and `dispatch_readiness.ready = false`.

## Why The Visible Claim Is False

Cancelled tasks are terminal. They should be absent from active-work buckets or
shown only in an explicit cancelled/history treatment.

Displaying cancelled tasks as `Waiting on you` or dispatchable tells the human
there is current work to perform when durable task truth says the work is no
longer active.

## Likely Root Cause

The backend state mapper does not treat `status: cancelled` as terminal before
deriving attention, owner, and dispatch readiness. `Task-0004` also appears to
fall into the same unmapped default branch as completed tasks, making it
dispatch-ready despite closure.

## Required Closure Proof

- `/api/v1/tasks` either omits cancelled tasks from active task readback or
  returns them as a terminal cancelled/history state.
- Cancelled tasks are not counted in `Needs you`, `Ready`, `Running`,
  `Sleeping`, or `Blocked` active-work buckets.
- Cancelled tasks are not dispatchable.
- REG-004 app-surface reconciliation proves the visible cancelled-task handling
  against `Tracking/Task-0004/TASK-STATE.json` and
  `Tracking/Task-0006/TASK-STATE.json`.

## 2026-04-27 REG-004 Retest

- [REG-004-20260427-005957.md](./Testing/REG-004-20260427-005957.md)
