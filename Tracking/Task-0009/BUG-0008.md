# BUG-0008: Planning Tasks Do Not Expose Their Codex Planning Thread

Status: open

## Summary

Planning-state tasks shown in the `Tasks` tab do not expose a Codex thread link
for the planning conversation that created or prepared the task.

## Visible Evidence

Human-provided dashboard screenshot on 2026-04-27:

- selected task: `Task-0011`
- visible state: `Waiting on you`
- visible artifacts: `TASK.md`, `TASK-STATE.json`, `PLAN.md`, `HANDOFF.md`,
  and `PLAN approval target`
- no visible `Open Thread`, `Open Live Thread`, `Open Planning Thread`, or
  equivalent Codex-thread action

I checked `Tracking/Task-0011` for thread, session, conversation, or Codex link
references and did not find a durable thread URI or session identifier in the
task docs.

## Why This Is Concerning

If a task is in planning, a human or agent likely had a thread or session where
the planning work started. If that thread exists, the `Tasks` surface should let
the human jump to it directly. If no planning thread exists or no durable link
was captured, the task is in an inconsistent or under-attributed state: the
task asks for planning approval without preserving the planning context that
would make approval trustworthy.

## Likely Root Cause

The task artifact model and/or backend `deep_context` readback does not require
or preserve a planning-thread launch target for pending planning tasks.

## Expected Behavior

- Planning tasks expose a durable planning-thread launch target when one
  exists.
- The dashboard renders that target with a clear action label such as
  `Open Planning Thread` or `Open Thread`.
- If no planning thread exists, the backend/task state marks that explicitly so
  the dashboard can show a real missing-context condition rather than silently
  omitting it.

## Required Closure Proof

- For a planning task such as `Task-0011`, durable state or backend readback
  includes either a Codex planning-thread target or an explicit
  no-thread-captured reason.
- The dashboard renders the thread target when present.
- If missing, the dashboard and/or task artifact records the missing planning
  context as a visible blocked/incomplete planning condition.
