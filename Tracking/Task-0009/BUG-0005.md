# BUG-0005: Completed Task Displays As Running With Pause Available

Status: closed

## Summary

The `Tasks` tab displays `Task-0008` as running and exposes a `Pause` action,
even though the authoritative durable task state says the task is complete and
in closure.

## Visible Evidence

Regression artifact:

- [REG-004-20260427-0048.md](./Testing/REG-004-20260427-0048.md)

Supporting artifacts:

- [overlay-summary.txt](./Testing/REG-004-20260427-0048/overlay-summary.txt)
- [overlay.png](./Testing/REG-004-20260427-0048/overlay.png)
- [frontend-mapped-tasks.json](./Testing/REG-004-20260427-0048/frontend-mapped-tasks.json)

The visible dashboard summary showed:

```text
tasks_running=01
```

The mapped frontend state identified that running row as:

```text
Task-0008 -> visible_label=Running, visible_bucket=running, Pause=True
```

## Durable/Backend Evidence

Durable source:

- [Task-0008 TASK-STATE.json](../Task-0008/TASK-STATE.json)

Durable truth:

```json
{
  "status": "complete",
  "phase": "closure",
  "current_gate": "closure",
  "validation": {
    "build": "passing",
    "unit_tests": "passing",
    "regression": "not_applicable"
  }
}
```

Backend `/api/v1/tasks` returned `Task-0008` with an active-looking run state:

- `state_envelope.state = running`
- `state_envelope.next_owner = backend_worker`
- frontend action mapping exposed `Pause` as allowed

## Why The Visible Claim Is False Or Unsupported

The durable task state says the task is complete. A completed task must not be
shown as active running work unless durable/backend truth also proves that an
active run still owns the task story and that the terminal task state is wrong
or superseded.

The dashboard currently gives the human a live-run control for work whose task
state says it is closed.

## Likely Root Cause

Backend task readback appears to prefer stale or synthetic active run state over
the authoritative `TASK-STATE.json` terminal state. The state merger needs a
clear precedence rule for terminal durable task states versus lingering run
records.

## Required Closure Proof

- `/api/v1/tasks` reconciles `Task-0008` so durable `status: complete` and
  `phase: closure` cannot appear as `running` without explicit superseding
  evidence.
- The dashboard does not show `Task-0008` in the `Running` bucket and does not
  expose `Pause` for it while durable state remains complete.
- REG-004 app-surface reconciliation proves the `Running` bucket and
  `Task-0008` row against both backend readback and
  `Tracking/Task-0008/TASK-STATE.json`.
