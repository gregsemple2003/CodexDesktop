# BUG-0001 Jobs Tab Interaction Regression

## Summary

The `Jobs` tab currently performs jobs-state refresh work as part of the tab click path. On the live app this causes visible hitching, and the Windows discovery path can surface extra shell windows during the interaction.

## Repro

1. Launch the real dashboard app.
2. Trigger the overlay with `Ctrl+Alt+Space`.
3. Click the `Jobs` tab.

## Observed

- the overlay hitches during the tab switch
- extra windows can appear while the tab is switching
- the tab styling also does not match the approved Stitch reference closely enough

## Expected

- clicking `Jobs` switches surfaces immediately
- no background refresh or reconcile work runs on tab click
- jobs refresh happens only when the user explicitly presses the refresh or reconcile button
- the tab shell matches the approved Stitch structure and weight more closely

## Suspected Cause

- `select_tab("jobs")` currently calls `refresh_jobs_data()` directly
- `refresh_jobs_data()` synchronously drives Windows discovery on the UI thread
- the jobs backend currently shells out through PowerShell, which can surface visible windows if not hidden explicitly

## Next Step

- remove automatic jobs refresh from the tab click path
- hide child shell windows for jobs discovery and apply operations
- restyle the tab shell against `Tracking/Task-0004/Design/STITCH-JOBS-TAB-0001/`
